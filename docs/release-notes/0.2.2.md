# v0.2.2

A patch on top of [v0.2.1](0.2.1.md): refinements to the mobile logs view for
better UX on narrow screens. No configuration or deployment changes -- upgrade in
place.

## What's fixed

- **"Block via" dropdown no longer overflows on mobile.** On narrow screens with
  long instance names, the block-target selector is now stacked with its label
  above a full-width select, so the page never scrolls sideways. The desktop
  inline layout is unchanged.
- **Domain row spans full width on mobile.** On phones, the domain now occupies
  its own full-width row instead of pairing with the query type, making long
  domains easier to read. The query type is hidden on mobile to keep the card
  focused; it remains visible on desktop.
- **Timestamps render in your local timezone.** Query log times are now shown in
  the timezone of the browser viewing them, rather than the server's timezone.
  The server continues to emit an unambiguous UTC value; client-side script
  reformats it to the viewer's local time on load and after each page update.
  No server-side timezone configuration needed.

## Container images

Multi-architecture images (`linux/amd64` and `linux/arm64`) are published to the
GitHub Container Registry:

```
ghcr.io/kenlasko/adguard-logcentral:0.2.2    # this release
ghcr.io/kenlasko/adguard-logcentral:0.2      # latest patch of the 0.2 line
ghcr.io/kenlasko/adguard-logcentral:latest   # newest release
```

Each image ships with a signed SBOM and build provenance attestation, is scanned
with Trivy before publication, and is signed with cosign keyless.

## Upgrading

Drop-in for v0.2.x: no new or changed environment variables, no schema, no
storage. Pull the new image (or move your `0.2` / `latest` tag) and restart.

See the [README](../../README.md) for full configuration, deployment
(docker run / compose / Kubernetes), and OIDC setup instructions.
