# v0.1.1

A UI-focused patch on top of [v0.1.0](0.1.0.md): sharper, more precise
searching, one-click filtering from any log row, a per-instance breakdown on the
stats page, and a brand refresh to **AdGuard-LogCentral**. No configuration or
deployment changes -- upgrade in place.

## What's new

- **Exact-by-default search with `*` wildcards.** A plain search term now matches
  the whole value only, so `192.168.1.2` no longer also matches `192.168.1.20`.
  Add `*` anywhere to widen the match: `192.168.1.2*` (prefix), `*.example.com`
  (suffix), `*ads*` (substring), or in the interior. Matching is
  case-insensitive and anchored to both ends of the client or domain. AdGuard's
  own unanchored substring search is used only as a coarse pre-filter to narrow
  the fetched candidates; the precise anchored match is applied to the merged
  page, so cursor pagination is undisturbed and no real match is ever dropped.
- **Click to filter.** Click any client or domain in a log row to instantly
  filter the view by that exact value.
- **Per-instance stats breakdown.** The stats page now includes a *Statistics by
  instance* table -- one row per instance showing its queries, blocked count,
  blocked percentage, and average processing time -- alongside the merged totals
  and the existing combined top-10 lists.

## Changed

- **Rebrand to AdGuard-LogCentral.** The page title and header brand are now
  **AdGuard-LogCentral** (previously "AdGuard Log Aggregator" / "AdGuard Logs").

## Container images

Multi-architecture images (`linux/amd64` and `linux/arm64`) are published to the
GitHub Container Registry:

```
ghcr.io/kenlasko/adguard-logcentral:0.1.1    # this release
ghcr.io/kenlasko/adguard-logcentral:0.1      # latest patch of the 0.1 line
ghcr.io/kenlasko/adguard-logcentral:latest   # newest release
```

Each image ships with a signed SBOM and build provenance attestation, is scanned
with Trivy before publication, and is signed with cosign keyless.

## Upgrading

Drop-in for v0.1.0: no new or changed environment variables, no schema, no
storage. Pull the new image (or move your `0.1` / `latest` tag) and restart.

See the [README](../../README.md) for full configuration, deployment
(docker run / compose / Kubernetes), and OIDC setup instructions.
