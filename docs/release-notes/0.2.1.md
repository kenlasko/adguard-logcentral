# v0.2.1

A UI patch on top of [v0.2.0](0.2.0.md): the phone layout of the query log is
reworked into a denser, header-free card, and blocking on mobile moves from a
per-row button to a long-press with a confirmation step. No configuration or
deployment changes -- upgrade in place.

## What's new

- **Redesigned mobile log rows.** Below 700px wide, each query log entry is now a
  compact two-column card with no repeated column headers, pairing the fields
  onto four lines: time and instance, client address and its friendly name,
  domain and query type, then result and elapsed time. Long domains wrap inside
  their column instead of pushing the page sideways. The desktop table layout is
  unchanged.
- **Long-press to block or unblock on mobile.** The per-row Block / Unblock
  button is hidden on phone-sized screens. Instead, press and hold a log row to
  open a confirmation dialog that spells out the domain and the target instance
  before applying the change. The rule is still applied to the instance chosen in
  **Block via**, and the action flips (Block vs Unblock) based on whether the
  entry was already blocked. The desktop inline button is unchanged.

## Fixed

- **"Block via" dropdown overflow on mobile.** On narrow screens the block-target
  selector could spill past the edge of the viewport. It now takes its own
  full-width row and its select shrinks to fit, so the page no longer scrolls
  sideways.

## Container images

Multi-architecture images (`linux/amd64` and `linux/arm64`) are published to the
GitHub Container Registry:

```
ghcr.io/kenlasko/adguard-logcentral:0.2.1    # this release
ghcr.io/kenlasko/adguard-logcentral:0.2      # latest patch of the 0.2 line
ghcr.io/kenlasko/adguard-logcentral:latest   # newest release
```

Each image ships with a signed SBOM and build provenance attestation, is scanned
with Trivy before publication, and is signed with cosign keyless.

## Upgrading

Drop-in for v0.2.0: no new or changed environment variables, no schema, no
storage. Pull the new image (or move your `0.2` / `latest` tag) and restart.

See the [README](../../README.md) for full configuration, deployment
(docker run / compose / Kubernetes), and OIDC setup instructions.
