# v0.2.0

A UI-focused release on top of [v0.1.1](0.1.1.md): the whole interface now works
on a phone, and search reaches the client's friendly name, not just its address.
No configuration or deployment changes -- upgrade in place.

## What's new

- **Responsive mobile layout.** The interface now adapts to phone-sized screens
  instead of overflowing sideways. Below 700px wide, each wide data table -- the
  query log, the per-instance stats, and the top-10 panels -- collapses into a
  stack of cards where every value keeps its column label beside it, and the top
  bar, filter toolbar, and stat grids reflow into a single tap-friendly column
  (full-width search box, and full-width Block / Unblock and Load more buttons).
  The desktop table layout above that width is unchanged.
- **Search by client name.** The search box now also matches the client's
  friendly name -- the name AdGuard shows in brackets beside the address -- so
  you can find a device by the name you know it by, not only its IP. It uses the
  same exact-by-default, `*`-wildcard, case-insensitive rules already applied to
  the domain and client address (`Kitchen-Tablet`, `Kitchen-*`, `*-TV`, and so
  on). As before, AdGuard's own search is used only as a coarse pre-filter -- its
  query log search already covers the client name -- and the precise anchored
  match is applied to the merged page, so cursor pagination is undisturbed and no
  real match is ever dropped.

## Container images

Multi-architecture images (`linux/amd64` and `linux/arm64`) are published to the
GitHub Container Registry:

```
ghcr.io/kenlasko/adguard-logcentral:0.2.0    # this release
ghcr.io/kenlasko/adguard-logcentral:0.2      # latest patch of the 0.2 line
ghcr.io/kenlasko/adguard-logcentral:latest   # newest release
```

Each image ships with a signed SBOM and build provenance attestation, is scanned
with Trivy before publication, and is signed with cosign keyless.

## Upgrading

Drop-in for v0.1.x: no new or changed environment variables, no schema, no
storage. Pull the new image (or move your `0.2` / `latest` tag) and restart.

See the [README](../../README.md) for full configuration, deployment
(docker run / compose / Kubernetes), and OIDC setup instructions.
